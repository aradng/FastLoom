# Internationalization (i18n)

Babel + Jinja2 + `gettext` translations driven by the request's `Accept-Language` header. Custom exceptions carry translatable messages and serialize to a uniform JSON shape.

**Symbols at a glance**

- `fastloom.i18n.base.CustomI18NException` — base class for translatable errors.
- `fastloom.i18n.base.SettingNotSet`, `DoesNotExist`, `OnlyAdminOrOwnerAllowed`, `OnlyOwnerAllowed` — built-in exceptions.
- `fastloom.i18n.handler.i18n_exception_handler` — registered automatically by the launcher.
- `fastloom.i18n.handler.get_template` — render a Jinja2 template with the active locale.
- `fastloom.i18n.handler.lang_dict` — quick multi-language dictionary literal.
- `fastloom.i18n.handler.parse_locale`, `set_locale` — locale parsing helpers.
- `fastloom.i18n.types.Languages` (`BASE`, `EN`, `FA`), `TranslatedValue`, `Translation`.

> The `Languages` enum is fixed at `BASE / EN / FA` for historical reasons (this codebase was first deployed for a Persian-speaking market). Treat `FA` as "the project's primary non-English locale slot" if your service targets a different language pair — adapt the `lang_dict` signature in a helper of your own.

## Locale layout

```
locale/
    en/LC_MESSAGES/messages.po
    <lang>/LC_MESSAGES/messages.po
templates/
    welcome.html
```

Both directories live at the project root (`Path.cwd() / "locale"` and `.../"templates"`). Compile `.po` to `.mo` with `pybabel compile -d locale` as part of your build.

## Custom exceptions

Subclass `CustomI18NException` with a `message` (passed through `gettext`) and an `error_code`:

```python
from gettext import gettext as _
from fastapi import status
from fastloom.i18n.base import CustomI18NException


class GroupLinkConflict(CustomI18NException):
    message = _("Cannot re-share groups. Conflicts: {offenders}")
    error_code = status.HTTP_400_BAD_REQUEST
    offenders: str

    def __init__(self, offenders: list[str]):
        self.offenders = ", ".join(offenders)
        super().__init__(self.message, self.error_code)
```

When you `raise GroupLinkConflict([...])`, the auto-registered handler returns:

```json
{
    "detail": "Cannot re-share groups. Conflicts: {offenders}",
    "message": "Cannot re-share groups. Conflicts: {offenders}",
    "message_tr": "Cannot re-share groups. Conflicts: a, b"
}
```

- `detail` and `message` are the **raw template** (no formatting, for stable machine matching).
- `message_tr` is the translated and formatted message, picked from the request's `Accept-Language` header.

### Built-in exceptions

| Class | HTTP code | Message template |
|-------|-----------|------------------|
| `SettingNotSet(name)` | 503 | `{name} not set` |
| `DoesNotExist(name)` | 404 | `{name} does not exist` |
| `OnlyAdminOrOwnerAllowed` | 403 | `Only admin or owner allowed to access` |
| `OnlyOwnerAllowed(resource, action)` | 403 | `Only owner of {resource} allowed to {action}` |

Use them directly:

```python
from gettext import gettext as _
from fastloom.i18n.base import DoesNotExist

raise DoesNotExist(_("User"))
```

## Translating outside of exception messages

`set_locale(locale)` installs the gettext translation table for the parsed language and returns the bound `gettext`. The exception handler calls it once per request; if you need translations inside the handler body, use `_()` (resolved at call time):

```python
from gettext import gettext as _

response = {"label": _("Welcome")}
```

`parse_locale` accepts any `Accept-Language` value and falls back to `"en"` on invalid input.

## Templates

```python
from fastloom.i18n.handler import get_template

html = get_template(
    "welcome.html",
    locale=request.headers.get("Accept-Language"),
    name="Alice",
)
```

`get_template`:

1. Parses the locale (default `en`).
2. Builds a Jinja2 environment over `Path.cwd() / "templates"` with the `i18n` extension enabled.
3. Loads `Translations` for the parsed language.
4. Renders the template with `gettext`, `lang`, and any `**kwargs` you pass.

Inside the template, use `{% trans %}...{% endtrans %}` and `{{ gettext("...") }}` as usual.

## `lang_dict`

A shortcut for storing multi-language labels on database records. The function signature is `lang_dict(fa: str, en: str | None = None)` — the first argument is stored both as `Languages.BASE` and `Languages.FA`; the second (if provided) is `Languages.EN`. If your project doesn't use the `FA` slot, pass the same string twice or wrap `lang_dict` in a project-specific helper.

```python
from fastloom.i18n.handler import lang_dict
from fastloom.i18n.types import Translation
from pydantic import BaseModel


class Tag(BaseModel):
    label: Translation = lang_dict("Hello world")               # same string in every slot
    headline: Translation = lang_dict("Bonjour le monde", "Hello world")
```

`Translation` serializes with `exclude_none_serializer` — keys whose values are `None` are dropped on output.
